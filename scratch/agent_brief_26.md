## Agent Brief

**Category:** enhancement
**Summary:** Migrate navigation menu to FluentNavMenu (v5) and implement collapsible layout width sizing

**Current behavior:**
The navigation panel has a static width of `250px` in `MainLayout.razor`'s `<FluentLayoutItem Area="@LayoutArea.Navigation" Width="250px">`.
The navigation menu in `NavMenu.razor` uses `<FluentNav Width="100%">` and `<FluentNavItem>`, which do not natively collapse or communicate expansion state. The hamburger menu toggles a boolean state `_menuExpanded` via `OnHamburgerOpened`, but the sidebar width remains static.

**Desired behavior:**
1. Migrate `NavMenu.razor` to use `<FluentNavMenu Collapsible="true" CustomToggle="true">` with `<FluentNavLink>` items instead of `<FluentNav>` and `<FluentNavItem>`.
2. Expose a parameter `Expanded` (boolean) and callback `ExpandedChanged` (EventCallback<bool>) in `NavMenu.razor` to support two-way binding of the navigation menu's expansion state.
3. In `MainLayout.razor`, bind the `<NavMenu />` component using `@bind-Expanded="_menuExpanded"`.
4. Update the `<FluentLayoutItem Area="@LayoutArea.Navigation">` in `MainLayout.razor` to set its width dynamically (e.g., `Width="@(_menuExpanded ? "250px" : "60px")"`).

**Key interfaces:**
- `NavMenu.razor` parameters:
  - `Expanded` (bool)
  - `ExpandedChanged` (EventCallback<bool>)
- `MainLayout.razor` layout bindings:
  - `<FluentLayoutItem Area="@LayoutArea.Navigation" Width="@(_menuExpanded ? "250px" : "60px")">`
  - `<NavMenu @bind-Expanded="_menuExpanded" />`

**Acceptance criteria:**
- [ ] `NavMenu.razor` is migrated to use `FluentNavMenu` (v5) and `FluentNavLink` components instead of `FluentNav`/`FluentNavItem`.

- [ ] In `MainLayout.razor`, the navigation menu width is dynamic: `250px` when expanded, and `60px` when collapsed.
