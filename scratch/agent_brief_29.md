**Summary:** Add more themes (Tokyo Night, Catppuccin, Solarized, SynthWave '84) and a theme selector dropdown

**Current behavior:**
The application only supports a binary light/dark mode toggle using `ThemeService.SwitchThemeAsync()` via a button in `MainLayout.razor`. There is no support for custom theme palettes or background overrides.

**Desired behavior:**
1. Support the following themes with their respective mode, brand color, and background:
   - **Tokyo Night** (Dark, brand: `#7aa2f7`, background: `#1a1b26`)
   - **Catppuccin Dark** (Dark, brand: `#c6a0f6`, background: `#1e1e2e`)

   - **Solarized Dark** (Dark, brand: `#268bd2`, background: `#002b36`)

2. In the header of `MainLayout.razor`, replace or supplement the current theme toggle button with a dropdown selector containing these theme options.
3. When a theme is selected:
   - Call `ThemeService.SetThemeAsync(...)` passing a `ThemeSettings` object configured with the theme's brand color and mode.
   - Use JS interop to set a `data-theme-name` attribute on the `<body>` element.
   - Persist the selected theme settings/name in `localStorage`.
4. In `wwwroot/app.css`, write CSS overrides targeting `body[data-theme-name="theme-slug"]` to define custom backgrounds and override layout design tokens (like `--neutral-fill-layer-rest`) for each theme.
5. In `App.razor`, update the `<script>` in `<body>` to read the persisted theme selection from `localStorage` on page load, setting the appropriate `data-theme` and `data-theme-name` attributes early to prevent flashing.

**Key interfaces:**

- `App.razor` (loading script)
- `wwwroot/app.css` (custom CSS token overrides)
