## Agent Brief

**Category:** enhancement
**Summary:** Implement native FluentUI drag-and-drop for cards between board columns

**Current behavior:**
Cards are rendered statically in columns on the Kanban board. Moving a card to another column requires opening the card dialog, selecting a different column from the dropdown, and saving. There is no direct drag-and-drop interaction.

**Desired behavior:**
1. Add a method `UpdateCardColumnAsync(Guid cardId, Guid columnId)` to `KanbanService.cs` that patches only the `columnid` property of the card via Data API Builder (DAB).
2. In `Board.razor`, wrap the main board canvas containing the columns inside a `<FluentDragContainer TItem="CardDetailDto" OnDropEnd="HandleDropEnd">`.
3. Wrap each column's card container in a `<FluentDropZone TItem="CardDetailDto" Id="@col.Id.ToString()" Droppable="_isAuthenticated" Draggable="false">`.
4. Wrap each individual card rendering inside a `<FluentDropZone TItem="CardDetailDto" Item="card" Draggable="_isAuthenticated" Droppable="false">`.
5. Implement `HandleDropEnd` to extract the card details from `args.Source.Item` and the target column GUID from `args.Target.Id`. If the column has changed, call `UpdateCardColumnAsync` and reload the board on success. If the service call fails, show an error message in `_errorMessage`.
6. Ensure drag-and-drop functionality is only active for authenticated users (`_isAuthenticated` is true).

**Key interfaces:**
- `KanbanService.cs`:
  - `Task<bool> UpdateCardColumnAsync(Guid cardId, Guid columnId)`
- `Board.razor` components:
  - `<FluentDragContainer TItem="CardDetailDto" OnDropEnd="HandleDropEnd">`
  - `<FluentDropZone TItem="CardDetailDto" Id="@col.Id.ToString()" Droppable="_isAuthenticated" Draggable="false">` (Column Drop Target)
  - `<FluentDropZone TItem="CardDetailDto" Item="card" Draggable="_isAuthenticated" Droppable="false">` (Card Drag Source)

**Acceptance criteria:**
- [ ] `KanbanService` contains `UpdateCardColumnAsync` method that successfully patches a card's column in the database.
- [ ] Board columns act as drop targets when the user is logged in, and cards act as drag sources.
- [ ] Unauthenticated visitors cannot drag cards, and columns do not accept drops for them.
- [ ] Dropping a card onto a column successfully updates its column in the database and reloads the board.
- [ ] Dropping a card onto another card within the same column or a different column correctly targets that column and works as expected.
- [ ] Rebuilding and running the app functions without runtime issues.

**Out of scope:**
- Ordering cards within the same column.
- Modifying card categories or tags via drag-and-drop.
